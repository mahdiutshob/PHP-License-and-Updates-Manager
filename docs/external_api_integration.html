Client/External API Integration

Once the main script is up and running we can now integrate LicenseBox with your PHP application, wordpress plugin/themes. using the external/client helper file for LicenseBox external API!

Getting Started

  1. Login to LicenseBox and add a new product from the menu.

  2. Create a new API Key of type 'External' from the General & API Settings Page.

  3. Go to Client Helper File generator from the Generate dropdown on the menu.

  4. Select Product, API Key, Verification Period (if you want to automatically verify license at choosen interval) and choose if you want to allow Envato Purchase codes or not, Then click Generate and copy the generated code.

  5. Create a new folder in your php application, wordpress plugin root and Save the generated file as whatevar name you want to use, we are going to use lb_helper.php and the folder name as includes. So when you are done with this step you should have an includes folder with lb_helper.php file.

  6. If you want to make any changes in the lb_helper.php file like changing of response messages etc then you should do that now. Some other things you would want to take a look at is $root_path and $license_file path they should be pointed to your application root or plugin root if you are using it in wordpress plugin/theme (if your includes folder which has the lb_helper.php file is in your application root then you don't have to make any changes)

  7. Once we are done with the external helper file, it should now be obfuscated. We are going to use 2 obfuscators together in this example, make sure you follow the same order.

    1. PHP Protect Webfuscator : Go to http://www.phpprotect.info/webfuscator and paste all the code from lb_helper.php file and click obfuscate, once it's done save the obfuscated code back in lb_helper.ph file

    2. FOPO : Go to http://www.fopo.com.ar/ and paste all the code from lb_helper.php file, once it's done save the obfuscated code back in lb_helper.ph file

      3. Some shared web hosting providers consider eval() based obfuscations (which FOPO does) as a threat. This may result in your webserver ports getting blocked so if your application will be used on a shared hosting and not on a vps or your own server or localhost then you should skip the FOPO obfuscation step.
      If you are planning to sell your application on Envato you should also not use FOPO as Envato does not allow too much code obfuscation.

      There are many other great free php obfuscation solution out there but if you want something really strong, Please use ioncube to obfuscate and encrypt your complete application once licensebox client API integration is done. ioncube is supported by most web hosting providers and it is an industry favourite when it comes to php code protection.

  8. We are now done with the lb_helper.php file, we should now include it in your application or plugin.

  9. If you want to only use the included sample php application installer, updater, activator or deactivator, Just copy the /install, /update, /activate or /deactivate folder respectively from LicenseBox examples folder from the downloaded .zip file and paste your database sql data dump in install/database.sql file (if using /install) and you are good to go. You don't need to do anything else, you can still use all the functions of lb_helper.php file in your application but you would need to require it at the top on every page where you wish to use it.

  10. If you want to use all or any LicenseBox API helper functions in your complete application, you should go to your application/plugin entry file (index.php file) or (innit.php file) or any other file which all other files of your application require/include.

  11. At the very top paste the following code:
    require 'includes/lb_helper.php';
    $api = new LicenseBoxAPI();
    lb_helper.php file on its own does nothing but it contains all the required function and you can call them however you want.

  12. Now you can call any available function from anywhere in your application, to learn about all the available functions of the external API and what they do with a live example, copy the external_api_examples.php file from licensebox examples folder in your application root and run it.

  13. For example you can verify license using verify_license() function and exit your application if license is invalid. verify_license() can be directly called without any parameters when a local .lic license file exist, this file is created on successful activation by activate_license() function, or you can run activate_license() function when user installs your application using included installer (/install) or you can use included activator (/activate) to just allow this functionality and not have a full installer.
    $res = $api->verify_license();
if($res['status']!=True){
die('Your license is invalid, please contact support');
}
  14. You can also verify license based on the choosen verification period.
    $res = $api->verify_license(true);
if($res['status']!=True){
die('Your license is invalid, please contact support');
}
  15. Lets look at an example for checking if updates are available or not.
    $res = $api->check_update();
if($res['status']){
echo "New ".$res['version']." update available.";
}

    16. There are a lot more ways in which you can use LicenseBox Client API in your application, see the external_api_examples.php file to learn more.

  16. Once you are done with all the changes in your entry file i.e index.php or innit.php, you should also obfuscate this file using the above mentioned method or anything else which you like.

    17. Using Included Examples

    Examples file (external_api_examples.php)

    You can check out the external_api_examples.php file, it has all the LicenseBox external/client API methods with their explanations and live examples. To enable live examples copy this file in your application root where there is includes folder which has lb_helper.php file.

    Activator (/activate folder)

    You can also use a pre-made script activator if you want to save time, just copy the activate folder from licensebox_api_examples folder in your application root, this script creates a local .lic file based on license code and client name, once user has activated the script or created .lic file you can check the licene using verify_license() function as explained above.

    Installer (/install folder)

    You can also use a pre-made script installer if you want to save time, just copy the install folder from licensebox_api_examples folder in your application root and add your sql data dump in the database.sql file.

    Updater (/update folder)

    You can also use a pre-made script updater if you want to save time, just copy the update folder from licensebox_api_examples folder in your application root, just make sure you have a $root_path in lb_helper.php pointing to your application root. NOTE: When you release any new update for your application you should also include the updated lb_helper.php file with $current_version changed to the new version in the update zip file.

    Deactivator (/deactivate folder)

    You can also use a pre-made script deactivator if you want to save time, just copy the deactivate folder from licensebox_api_examples folder in your application root, this script deactivates and removes the local .lic file and it marks its activation as inactive on the server, So that the client can activate the same license on some other domain.

    Wordpress Sample Plugin (/wordpress-sample-plugin folder)

    You can also checkout the included sample wordpress plugin to see how licensebox can be integrated with a wordpress plugin, copy the plugin folder and add lb_helper.php file in the includes folder then create a zip file and install it in your wordpress site. This plugin simply displays some dummy text in posts footer when the license is active.
